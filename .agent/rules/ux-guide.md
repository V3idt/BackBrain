- **Never insult the user's intelligence**: Don't tell them what to do. Make the interface so intuitive that it's clear.
- **Provide subtle guidance**: Use small, visual demos or guides only when a user reaches a specific feature for the first time.
- **"Just Works" Philosophy**: Aim for a frictionless experience where software is secure and functional without the user needing to spend hours debugging.
